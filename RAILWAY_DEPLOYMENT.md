# Railway Deployment Guide for Laravel

Your project is properly configured for a seamless deployment on [Railway](https://railway.app/). Railway uses **Nixpacks** to automatically detect your Laravel app, configure Nginx/PHP, and compile your frontend assets natively.

Follow these steps to deploy:

### 1. Create a Project on Railway
1. Go to your Railway dashboard and click **New Project**.
2. Select **Deploy from GitHub repo** and choose your repository.
3. The initial deployment might fail if your database and environment variables aren't set up yet. Don't worry, proceed to the next steps.

### 2. Provision a Database
If your app uses a database:
1. In your Railway Project dashboard, click **New > Database** and select **MySQL**.
2. Wait a few seconds for the database to provision.

### 3. Configure Environment Variables
1. Click on your Laravel Service (the one deployed from your GitHub repo).
2. Go to the **Variables** tab. Add the following necessary variables:
   - `APP_KEY`: Check your local `.env` and copy its value (it typically starts with `base64:...`). Ensure this is never lost.
   - `APP_ENV`: `production`
   - `APP_DEBUG`: `false`
   - `APP_URL`: The public-facing domain generated by Railway (e.g., `https://your-service.up.railway.app`)
   - `LOG_CHANNEL`: `stderr` *(This routes Laravel logs to the Railway dashboard so you can view them easily in the 'Logs' tab).*
3. **Database Configuration**: If you created a MySQL database addon, Railway provides specific database variables internally. Map Laravel's variables to Railway's variables like this:
   - `DB_CONNECTION`: `mysql`
   - `DB_HOST`: `${MYSQLHOST}`
   - `DB_PORT`: `${MYSQLPORT}`
   - `DB_DATABASE`: `${MYSQLDATABASE}`
   - `DB_USERNAME`: `${MYSQLUSER}`
   - `DB_PASSWORD`: `${MYSQLPASSWORD}`

### 4. Setup the Release Command (Migrations)
To automatically run your database migrations when deploying new code:
1. In your Laravel service dashboard, go to the **Settings** tab.
2. Scroll down to the **Deploy** section.
3. In the **Release Command** input field, add:
   ```bash
   php artisan migrate --force
   ```
4. This ensures that every time you push to your repository, your database is safely migrated *before* the new version of your project goes live.

### 5. Finalize Deployment
1. Go to your service's **Deployments** tab. If the latest deployment failed, click the three dots on the deployment and click **Redeploy**.
2. Your app should now build successfully, run migrations in the release phase, and deploy live!

> **Note:** We added `$middleware->trustProxies(at: '*');` in `bootstrap/app.php`. This tells Laravel to trust the headers passed by Railway's edge proxy, which guarantees that helpers like `url()` output secure `https://` URLs correctly without any further configuration.
